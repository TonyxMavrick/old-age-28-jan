1. Our Commitment to Accessibility

GoldenHaven Senior Care believes that everyone, regardless of ability, should have equal access to information and services. Digital accessibility is not just a legal requirementâ€”it's a moral imperative that aligns with our core mission of dignity and respect for all individuals.

We are dedicated to making our website accessible to the widest possible audience, including:

  • People with visual impairments
  • People with hearing impairments
  • People with motor/mobility impairments
  • People with cognitive disabilities
  • Older adults with age-related limitations
  • People using assistive technologies

2. Conformance Standards

2.1 WCAG 2.1 Level AA Compliance

Our website strives to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA. These guidelines explain how to make web content more accessible for people with disabilities and more user-friendly for everyone.

2.2 Four Principles of Accessibility

The WCAG guidelines are organized around four principles (POUR):

  • Perceivable: Information must be presentable to users in ways they can perceive
  • Operable: Interface components must be operable by all users
  • Understandable: Information and operation must be understandable
  • Robust: Content must be robust enough to work with current and future technologies

2.3 Additional Standards

We also follow best practices from:

  • Section 508 of the Rehabilitation Act
  • Americans with Disabilities Act (ADA) Title III
  • WAI-ARIA (Accessible Rich Internet Applications)

3. Accessibility Features

Our website includes the following accessibility features:

Keyboard Navigation

All interactive elements can be accessed and operated using only a keyboard. Use Tab to navigate, Enter to activate, and Escape to close dialogs.

Screen Reader Support

Semantic HTML and ARIA labels ensure compatibility with screen readers like JAWS, NVDA, and VoiceOver.

Text Alternatives

All images include descriptive alt text. Decorative images are marked appropriately for screen readers to skip.

Clear Structure

Logical heading hierarchy (H1-H6) and semantic HTML elements provide clear document structure.

Color Contrast

Text and interactive elements meet WCAG AA contrast ratio requirements (4.5:1 for normal text, 3:1 for large text).

Resizable Text

Text can be resized up to 200% using browser zoom without loss of functionality or content.

Focus Indicators

Clear visual focus indicators show which element has keyboard focus at all times.

Skip Navigation

Skip-to-content links allow keyboard users to bypass repetitive navigation and jump directly to main content.

Consistent Navigation

Navigation is consistent across all pages, making it easier to learn and use the site.

Form Labels

All form fields have clear, descriptive labels and instructions. Error messages are specific and helpful.

Readable Typography

We use clear, readable fonts with appropriate line spacing and text alignment.

No Time Limits

Users can complete forms and read content at their own pace without time restrictions.

Our website is designed to work with the following assistive technologies:

4.1 Screen Readers

  • JAWS (Job Access With Speech)
  • NVDA (NonVisual Desktop Access)
  • VoiceOver (macOS and iOS)
  • TalkBack (Android)
  • Narrator (Windows)

4.2 Browser Accessibility Features

  • Built-in zoom and text resizing
  • High contrast modes
  • Reader modes
  • Voice control (Dragon NaturallySpeaking, Voice Control)

4.3 Operating System Accessibility

  • Windows Accessibility Features
  • macOS Accessibility Options
  • iOS Accessibility Settings
  • Android Accessibility Suite

5. Known Limitations

Despite our best efforts, some content may not yet be fully accessible. We are aware of the following limitations:

5.1 Third-Party Content

Some embedded content from third-party providers (such as maps, videos, or social media) may not be fully accessible. We are working with these providers to improve accessibility.

5.2 PDF Documents

Older PDF documents may not be fully accessible. We are in the process of updating these documents. If you need an accessible format, please contact us.

5.3 Legacy Content

Some older pages or archived content may not meet current accessibility standards. We are systematically reviewing and updating this content.

6. Ongoing Improvements

Accessibility is an ongoing effort. We are committed to:

6.1 Regular Testing

  • Automated accessibility testing using industry-standard tools
  • Manual testing with screen readers and keyboard navigation
  • User testing with people who use assistive technologies
  • Regular audits by accessibility experts

6.2 Staff Training

  • Ongoing training for web developers and content creators
  • Accessibility best practices incorporated into our workflows
  • Regular updates on evolving accessibility standards

6.3 Technology Updates

  • Monitoring new accessibility technologies and standards
  • Implementing improvements as they become available
  • Testing with the latest assistive technology versions

7.1 What to Include in Your Report

When reporting an accessibility issue, please include:

  • The web page or content where you encountered the issue
  • A description of the problem
  • The assistive technology you were using (if applicable)
  • Your browser and operating system
  • Any steps to reproduce the issue

7.2 Our Response

We strive to respond to accessibility feedback within 2 business days. We will work with you to understand the issue and provide an accessible alternative while we implement a permanent solution.
